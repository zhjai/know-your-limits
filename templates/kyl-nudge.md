When a small/cheap model is doing a long task here, follow **know-your-limits**: escalate to a
senior model (via agent-arena) at the START of a substantial/risky task (plan review), before any
irreversible action (schema/migration/delete/deploy/auth/new-dep), when the same error survives 2
fix attempts, and before declaring a large/risky diff done. Don't gate this on "feeling unsure" —
use the objective tripwires. If the senior can't resolve it, ask the human; don't loop.
